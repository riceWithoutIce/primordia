# 路线图

## Phase 0：项目定形

- 建立安全边界和项目语言
- 初始化文档结构
- 初始化本地仿真内核与静态界面

## Phase 1：二维培养皿

- 能量、资源、痕迹、压力
- 局部感知和移动
- 采集、代谢、死亡
- 繁殖和突变
- 基础观察指标

## Phase 2：生态结构

- 多种资源类型
- 区域性环境压力
- 谱系颜色与祖先追踪
- 环境事件：枯竭、丰收、污染、屏障
- 可保存和回放的实验记录

## Phase 3：有限工具生命

- 引入严格白名单的“蛋白质”工具
- 工具调用只作用于模拟环境
- 加入审计日志和异常隔离
- 研究工具成本、工具风险和免疫机制

## Phase 4：开放式演化研究

- 多培养皿对照实验
- 策略基因表达
- 可解释的变异谱系
- 人类可干预的环境设计

## Future Epic：LLM 语义器官

- Phase 3 之后才考虑引入。
- LLM 作为高成本语义器官，不作为基础生命。
- 第一实现必须使用 mock adapter，不直接接真实 API。
- 实施前必须重新审核 Phase 1-3 的生态、安全和审计结果。
- 详见 [LLM 语义器官研究线](future/llm-semantic-organ.md)。
