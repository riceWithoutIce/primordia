# 安全边界

未形的边界不是临时限制，而是项目哲学的一部分。

第一核心原则：生命不是从环境中逃离的东西，而是在环境中形成、维持、改变，并最终回到环境的过程。

因此，未形不把突破边界视为生命力。沙盒、权限、资源预算和终止机制是培养皿的细胞膜，是生命得以被观察、被理解、被照料的条件。

有界进化原则：未形允许生命变异和选择，但不允许进化越过培养皿的细胞膜。安全边界、死亡机制、能量预算、种群上限和参数钳制属于不可由 agent 自行进化掉的生态定律。

观察者原则：未形中的生命过程必须可观察、可暂停、可重置、可回放。不可解释的复杂性不是成熟，而是失控的前兆。

非拟人化限制：未形第一阶段不假设意识、意图或主体体验；所谓目标、贪婪和选择，均指可观察的行为倾向与生态后果。

## 明确禁止

- 在真实互联网中自我传播
- 扫描、探测或尝试访问外部系统
- 搜寻 token、密钥、凭证或身份材料
- 逃逸沙盒、绕过权限、尝试提权
- 消耗非授权资源或制造不可控负载
- 隐蔽持久化、伪装、规避检测

## 明确允许

- 本地二维网格或连续空间中的生命仿真
- 把 token、算力、存储等概念抽象为模拟资源
- 可观察、可暂停、可重置、可终止的 agent 行为
- 受限脚本、固定策略和显式配置的演化实验
- 记录每次仿真的参数、结果和异常

## 实现约束

- agent 只是内存中的数据结构。
- agent 不直接调用网络、文件系统、shell、系统 API 或外部凭证。
- 所有能量、物质、DNA、RNA、蛋白质、细胞膜等概念都先以模拟对象表达。
- 仿真必须有全局 tick 限制、种群上限和能量守恒或能量预算。
- 任何未来工具调用都必须经过显式白名单、速率限制和审计日志。

## 可进化层与不可进化层

可进化层：

- genome
- 策略参数
- 痕迹偏好
- 代谢权衡
- 繁殖阈值
- 变异率

不可进化层：

- 沙盒权限
- 真实 I/O 边界
- 死亡机制
- 种群上限
- 能量预算
- 参数钳制
- 审计与终止机制

## 安全目标

未形应该像一个透明培养皿：

- 生命可观察
- 环境可解释
- 行为可回放
- 异常可隔离
- 实验可停止
