# 安全边界

未形的边界不是临时限制，而是项目哲学的一部分。

## 明确禁止

- 在真实互联网中自我传播
- 扫描、探测或尝试访问外部系统
- 搜寻 token、密钥、凭证或身份材料
- 逃逸沙盒、绕过权限、尝试提权
- 消耗非授权资源或制造不可控负载
- 隐蔽持久化、伪装、规避检测

## 明确允许

- 本地二维网格或连续空间中的生命仿真
- 把 token、算力、存储等概念抽象为模拟资源
- 可观察、可暂停、可重置、可终止的 agent 行为
- 受限脚本、固定策略和显式配置的演化实验
- 记录每次仿真的参数、结果和异常

## 实现约束

- agent 只是内存中的数据结构。
- agent 不直接调用网络、文件系统、shell、系统 API 或外部凭证。
- 所有能量、物质、DNA、RNA、蛋白质、细胞膜等概念都先以模拟对象表达。
- 仿真必须有全局 tick 限制、种群上限和能量守恒或能量预算。
- 任何未来工具调用都必须经过显式白名单、速率限制和审计日志。

## 安全目标

未形应该像一个透明培养皿：

- 生命可观察
- 环境可解释
- 行为可回放
- 异常可隔离
- 实验可停止
