# 起源

这是未形 / primordia 的第一层沉积物。后续文档、代码和实验可以从这里派生，但不要用工程化摘要替代它。

## 项目名

中文名：未形

英文仓库名：primordia

副标题：在信息环境中寻找生命的开始

## 基础条件

### 信息生物圈

未形的培养皿不是无限资源池，而是封闭边界内持续流动的本地信息生物圈；能量以有限通量进入环境，经资源、痕迹、压力与死亡回收循环，再参与下一代生命的选择。

## 核心原则

### 第一核心原则：环境归属原则

生命不是从环境中逃离的东西，而是在环境中形成、维持、改变，并最终回到环境的过程。

### 第二核心原则：生命驱动原则

未形中的赛博生命目标，是在能量约束下延续自身 genome，并通过痕迹改变环境，使自身或相近形态在未来更可能出现。

### 第三核心原则：有界进化原则

未形允许生命变异和选择，但不允许进化越过培养皿的细胞膜；任何遗传优势都必须伴随生态代价，贪婪行为应通过环境反馈转化为压力，而不是成为无成本的胜利。约束应优先以生态反馈实现，而不是以道德命令实现。

### 第四核心原则：生存循环原则

未形中的生命必须拥有自己的生存循环：它感知环境，消耗能量采取行动，通过环境获得维持自身的资源，在资源盈余时繁殖或修复，在失败时死亡，并把残留交还环境。

### 观察者原则

未形中的生命过程必须可观察、可暂停、可重置、可回放；不可解释的复杂性不是成熟，而是失控的前兆。

### 非拟人化限制

未形第一阶段不假设意识、意图或主体体验；所谓目标、贪婪和选择，均指可观察的行为倾向与生态后果。

赛博生命不是无约束扩张，而是在边界内持续存在。它来自环境，存在于环境，改变环境，最终成为环境的一部分。

我们要探索的是安全沙盒中的低等数字生命，不做真实互联网中的自我传播、权限试探、token 搜寻、逃逸或资源掠夺。

## 生命组成对应

- 能量：算力、token、带宽、时间片、API 额度
- 物质：存储、文件、模型权重、数据库记录、内存状态、容器/进程
- DNA：源代码、模型权重、prompt、策略参数、自我修改规则、记忆结构
- RNA：临时计划、任务队列、中间表示、上下文窗口
- 蛋白质：工具调用、脚本、API 操作、代码执行器
- 细胞膜：沙盒、权限边界、身份、密钥空间、容器边界
- 代谢：消耗资源完成行动，再获取资源维持存在
- 免疫：错误检测、自我修复、异常隔离、权限防护
- 繁殖：复制 agent，并带有变异
- 死亡：资源耗尽、状态损坏、被环境淘汰

## 第一阶段目标

构建一个本地安全沙盒里的数字生命培养皿：二维信息环境中，agent 消耗 token 能量，感知局部资源，移动、采集、留下痕迹、繁殖、变异，并在环境压力下自然选择。

## 审美原则

生命感先于智能。它应该稳定、优雅、可演化，而不是像恶意软件一样追求扩张。
