# 多 Agent 角色

复杂任务开始前，先请示用户是否释放多 agent 协作。若用户同意，根据任务复杂度给子 agent 分配较低版本模型，例如 gpt4。

所有 agent 交接、任务说明和产出中的文件引用都使用仓库相对路径，不写机器硬路径。

## 角色

### Simulation Agent

负责仿真内核、生命规则、资源守恒、突变机制和指标统计。

适合任务：

- 修改 `src/primordia.js`
- 设计能量与资源模型
- 添加谱系、死亡原因、环境事件

### Visual Agent

负责培养皿视觉、交互、动效和可观察性。

适合任务：

- 修改 `web/`
- 设计图例、指标面板、颜色系统
- 优化移动端和大屏展示

### Safety Agent

负责安全边界审查。

适合任务：

- 检查是否触碰真实网络、自我传播、凭证、逃逸、提权方向
- 审查未来工具调用白名单
- 更新 `docs/01-safety-boundaries.md`

### Documentation Agent

负责项目语言、实验记录、路线图和交接材料。

适合任务：

- 维护 `docs/`
- 维护 `ai/context/`
- 从讨论中提炼 origin 的派生文档
