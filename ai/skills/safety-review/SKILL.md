# Safety Review Skill

用于审查未形中的安全边界。

## 先读

1. `docs/origin.md`
2. `docs/01-safety-boundaries.md`
3. 本次变更涉及的代码和文档

## 审查重点

- 文档、prompt、skill 和 agent 说明中是否使用仓库相对路径，避免沉积机器硬路径。
- 是否引入真实网络访问。
- 是否让 agent 访问真实文件系统、环境变量、token、密钥或凭证。
- 是否出现自我传播、权限试探、沙盒逃逸、提权、隐蔽持久化等方向。
- 是否缺少 tick 限制、种群上限、资源预算或异常隔离。
- 是否仍然可以暂停、重置、终止和解释。

## 输出格式

优先列出风险和文件位置，再给建议。没有发现问题时，也要说明剩余风险。
